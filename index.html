Run Android games natively on Nintendo Switch

Android Horizon is a compatibility/translation layer that runs Android NDK games directly on the Switch's real operating system — Horizon OS. No Android OS, no emulator, no VM.

Download latest release Read the docs
Please read before trying this: this project is in extremely early stages. Approximately 99.8%+ of the code was written by Anthropic's Claude AI, working with a non-developer author who handles hardware testing and direction. Only one game has ever actually been run against this project. Use entirely at your own risk — see the compatibility list for the honest current status.

What is this?

Think of it like Wine on Linux, but for Android games on a game console — not emulation, but translation. The game's ARM64 machine code runs directly on the Switch's Tegra X1 processor, the same chip Android phones use. A thin shim layer fakes just enough of the Android runtime — libc, OpenGL ES, JNI, asset management — that the game's native .so library doesn't notice it isn't running on Android.

Current focus

  • Simple, old, 2D Android games
  • Ship an arm64-v8a native library
  • OpenGL ES 2.0/3.0 rendering
  • No Google Play Services dependency
  • Local saves only, no network requirement

What's working today

  • Full ELF loading, relocation & linking, zero unresolved symbols
  • Real libnx threads, mutexes, condvars behind pthreads
  • Real touch input driving the game's own native handlers
  • Real audio (OGG/MP3/Opus/FLAC) via a custom SDL2_mixer backend
  • A locked 60 fps in normal play, persistent saves

Architecture

Android Horizon is split into two pieces that chain-load into each other, rather than one monolithic binary:

AndroidHorizonNX

The launcher/picker. Scans for APKs, tags each by native ABI, and chain-loads the right engine on launch. No ELF loader or JNI shim of its own.

AHNX-Translation-Core

The engine. ELF loading, JIT, the JNI/Bionic compat layer, audio, sensors — everything that actually runs a game.
